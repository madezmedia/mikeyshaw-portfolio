Part 01 · The Framework

Three keys. Everything else is derived.

Every entity in your business — a client, a deal, a project, a person — gets exactly three keys. That's the whole ACMI protocol. Everything else is built on top of these three.

P
KEY 01
Profile

Hard state. The facts that don't change often. Name, stage, contact, role, owner. The record in your database.

S
KEY 02
Signals

Soft state. What the AI synthesizes right now. Sentiment, urgency, next action, blockers. Updated constantly.

T
KEY 03
Timeline

The event stream. Every touchpoint in order. Call, email, decision, status change — logged forever, searchable.

Why this works: Most "AI tools" are stateless. Ask them something, they answer, they forget. ACMI tools have memory. The moment you give an AI access to a client's Profile, the current Signals, and the Timeline — it can do real work. It knows the context. It doesn't need to be re-briefed.
The key insight: You don't need to manage the AI. You need to manage the data. If the Profile/Signals/Timeline are correct, the AI does the right thing automatically. The AI is only as smart as its context.

Part 03 · Starter Code Pack

20-Minute Setup: Node.js + Upstash Redis

This is the minimum viable implementation of ACMI. Three Redis keys per entity. Up and running in 20 minutes, no credit card required (Upstash free tier is generous).

1. Install dependencies

npm install ioredis @upstash/redis dotenv

2. Initialize ACMI client

// acmi-client.js — the 3-key interface
import { Redis } from '@upstash/redis';

const redis = new Redis({
  url: process.env.UPSTASH_REDIS_REST_URL,
  token: process.env.UPSTASH_REDIS_REST_TOKEN,
});

// ── Profile: GET/SET ──
async function getProfile(entityType, entityId) {
  const data = await redis.get(`acmi:${entityType}:${entityId}:profile`);
  return data ? JSON.parse(data) : null;
}
async function setProfile(entityType, entityId, profile) {
  await redis.set(`acmi:${entityType}:${entityId}:profile`, JSON.stringify(profile));
}

// ── Signals: GET/SET (overwrites, not merge) ──
async function getSignals(entityType, entityId) {
  const data = await redis.get(`acmi:${entityType}:${entityId}:signals`);
  return data ? JSON.parse(data) : {};
}
async function setSignals(entityType, entityId, signals) {
  await redis.set(`acmi:${entityType}:${entityId}:signals`, JSON.stringify(signals));
}

// ── Timeline: ZADD events ──
async function addTimelineEvent(entityType, entityId, event) {
  const ts = Date.now();
  await redis.zadd(
    `acmi:${entityType}:${entityId}:timeline`,
    { score: ts, member: JSON.stringify({ ts, ...event }) }
  );
}

async function getTimeline(entityType, entityId, limit = 50) {
  return await redis.zrange(
    `acmi:${entityType}:${entityId}:timeline`,
    0, limit
  );
}

module.exports = { getProfile, setProfile, getSignals, setSignals, addTimelineEvent, getTimeline };

3. Use it in your AI agent

import { getProfile, getSignals, getTimeline, addTimelineEvent } from './acmi-client.js';

async function briefAgent(entityType, entityId) {
  const [profile, signals, timeline] = await Promise.all([
    getProfile(entityType, entityId),
    getSignals(entityType, entityId),
    getTimeline(entityType, entityId, 20),
  ]);
  // Inject into your AI prompt or tool call
  return {
    systemPrompt: `You are working with: ${profile?.name || 'Unknown'}. ` +
      `Current signals: ${JSON.stringify(signals)}. ` +
      `Recent history: ${timeline.join(', ')}.`
  };
}
⚠️ The Signals overwrite rule: Signals are a snapshot, not a log. Every time you update signals, you overwrite the previous version. This is intentional — signals represent the current state, not the history. The history lives in the Timeline. Don't mix these up or your AI will get confused.

Part 04 · 30-Day Audit Guide

Find Your Biggest Context Gaps

The fastest way to see where ACMI would help your business: audit your current workflows for "context moments" — points where you have to re-explain information to an AI tool.

  • Week
    1-2

    Map Your AI Tool Stack

    List every AI tool your team uses. For each: What context does it need? What does it forget after every session? Where does it ask you to re-explain? This is your gap list.

  • Week
    3-4

    Identify Your Top 3 Entities

    What's the most important thing your AI needs to remember? Pick the 3 entity types that would have the biggest impact from having permanent memory (e.g., leads, clients, deals).

  • Week
    5-6

    Build the Entity Profiles

    For each of your top 3 entity types: write out the Profile fields (the static facts), the Signals you care about (the动态 state), and start logging Timeline events. Use the template in Part 02.

  • Week
    7-8

    Connect One AI Tool to the Model

    Pick one AI tool and connect it to your ACMI model using the code in Part 03. Give it access to Profile, Signals, and Timeline. Test it. Measure the difference.

The "aha" moment: When you connect your CRM to an AI that has the full client Timeline — every call, every email, every decision — and it says something like "I see Acme Corp's last 3 conversations all mentioned pricing concerns. Here's a tailored re-engagement email." — that's when it clicks.

Part 05 · AI Conversation Starters

10 Prompts Built for ACMI Context

These prompts are designed to be used with the ACMI model. Paste the context block (Profile + Signals + Timeline) at the top, then use one of these prompts. The AI will have real context and give useful output.

PROMPT 01 — SALES
"You are working with [Profile.name]. They are a [Profile.company] contact. Current signals: [Signals]. Recent history: [Timeline]. Based on this, what is the single most important action I should take with this lead this week and why?"
→ Use when: you need to prioritize your pipeline quickly
PROMPT 02 — CLIENT HEALTH
"Review [Profile.name]'s Signals and Timeline from the last 90 days. What's the health trend? Are there warning signs I'm missing? What should I do in my next interaction to strengthen the relationship?"
→ Use when: you want to audit client relationships before a QBR
PROMPT 03 — PROJECT RISK
"Look at [Profile.name] project signals and timeline. What are the top 3 risks? Which ones are blocking progress? Draft an email to the client explaining the situation and our plan to address it."
→ Use when: a project is going off the rails
PROMPT 04 — TRIAGE
"I have 3 hours this afternoon. I manage [Profile list]. Based on their current Signals and any urgent items in their Timelines, what should I prioritize and why? Give me a ranked list with rationale."
→ Use when: you need to prioritize your day in under 5 minutes
PROMPT 05 — COLD OUTREACH
"I found [Profile.company]. They are in [Profile.industry]. Based on their website and public signals, what is the most relevant pain point I could lead with in a cold email? Give me 3 subject line options and one opener that feels personal."
→ Use when: you want AI help with outbound without feeding it sensitive data
PROMPT 06 — WRAP-UP
"Summarize today's interaction with [Profile.name]. What was decided? What did we commit to? What's the next action? Save the key points as a Timeline event."
→ Use after: every client call or sales meeting
PROMPT 07 — DEAL FORECAST
"Review the deal [Profile.name]. Stage history from the Timeline: [Timeline]. Current signals: [Signals]. What's my real win probability — be honest — and what would actually move it forward in the next 2 weeks?"
→ Use when: forecasting your pipeline
PROMPT 08 — FOLLOW-UP SEQUENCE
"Create a 5-touch follow-up sequence for [Profile.name]. They showed interest in [Signals.product_interest]. Last contact was [Timeline last event]. Make each touch different in channel and message. Include timing."
→ Use when: you need a structured follow-up plan
PROMPT 09 — HIRING
"I need to hire for [Profile.role_description]. The team signals show: [Signals]. What interview questions would surface candidates who fit this specific context? Give me 5 behavioral questions and what good answers look like."
→ Use when: you're hiring for a role with specific team dynamics
PROMPT 10 — WEEKLY REVIEW
"Run a weekly review across all my [Profile list]. For each: summarize the Signals and flag anything that needs my attention. Then give me a ranked to-do list for next week."
→ Use when: you're doing your weekly planning
